1|# Agent 辅助学习工具指南
     2|
     3|> 宗旨：找到那些**放大你的思考而非替代你的思考**的工具
     4|> 筛选标准 —— 一个好工具应该让你在用它之后比用之前更理解问题，而不是更少
     5|
     6|---
     7|
     8|## 核心筛选原则
     9|
    10|| 通过 ✅ | 淘汰 ❌ |
    11||---|---|
    12|| 你定义问题和方向，工具加速执行 | 工具替你做决策，你只负责确认 |
    13|| 你主动输出，工具做结构化反馈 | 工具输出成品，你被动接收 |
    14|| 工具帮你看到盲区，但你自己走到答案 | 工具直接给答案，跳过了你的思考路径 |
    15|| 使用工具后你对该领域的理解加深了 | 使用工具后你对该领域的依赖加深了 |
    16|
    17|### 检查清单（选工具前问自己）
    18|
    19|```
    20|□ 这个工具是我在用它，还是它在用我？
    21|□ 用它之后，我是否比之前更理解这个问题的本质？
    22|□ 如果这个工具明天消失，我能否独立完成这件事？
    23|□ 它是在放大我的能力，还是在替代我的能力？
    24|```

    28|## 一、学习型 MCP 服务器推荐
    29|
    30|### 1.1 笔记与知识管理（构建你的第二大脑）
    31|
    32|#### Obsidian MCP（⭐4.1k / coddingtonbear / MarkusPfundstein）
    33|
    34|```
    35|用途：把你的笔记库变成 Agent 可读可写的知识系统
    36|核心能力：创建/读取/搜索/编辑 Markdown 笔记，管理标签和 frontmatter
    37|```
    38|
    39|**正确用法（高参与度）**：
    40|
    41|```
    42|Sprint：我刚学了 MCP 的 Transport 机制，在 Obsidian 建个笔记，
    43|把我理解的 stdio vs HTTP 对比写下来。然后让 Agent 帮我补充遗漏维度。
    44|
    45|你：帮我在 Obsidian 中搜索我已有的 MCP 相关笔记，
    46|找出"我理解有偏差"的地方。
    47|→ 先自己写笔记，用 Obsidian MCP 做交叉验证和盲区扫描
    48|```
    49|
    50|**错误用法（低参与度）**：
    51|
    52|```
    53|你：帮我把今天学的 MCP 整理成笔记存到 Obsidian。
    54|→ Agent 替你做了知识提炼，你没经历"用自己的话重述"这一步
    55|```
    56|
    57|#### Notion MCP（官方，⭐4.5k）
    58|
    59|```
    60|用途：结构化知识管理，适合项目管理型学习
    61|```
    62|
    63|**正确用法**：
    64|
    65|```
    66|你：帮我从 Notion 的学习计划数据库中列出这周标记为"模糊"的概念。
    67|然后我一个个解释给你听，你帮我判断是不是真的理解了。
    68|→ 你自己定义"模糊"的标准，Agent 只做检索和反馈
    69|```
    70|
    71|#### basic-memory（⭐3.5k）
    72|
    73|```
    74|用途：AI 对话的持久记忆系统，构建跨会话的知识基础
    75|```
    76|
    77|**正确用法**：
    78|
    79|```
    80|你：把今天的 3 个关键洞见存到 basic-memory。
    81|明天开始前，先让我回忆昨天学到了什么，
    82|然后你再取出记录做对比。
    83|→ 先主动回忆，再用记忆系统验证
    84|```

    88|### 1.2 代码审查与质量检查（让 Agent 做你的导师）
    89|
    90|#### SonarQube MCP（官方，⭐600）
    91|
    92|```
    93|用途：代码质量和安全分析
    94|```
    95|
    96|**正确用法（高参与度）**：
    97|
    98|```
    99|流程：
   100|1. 你自己写完代码
   101|2. 自己 review 一遍，标记你知道的问题
   102|3. 用 SonarQube MCP 跑静态分析
   103|4. 对比"你自己发现的问题" vs "SonarQube 发现的问题"
   104|5. 对于你没发现的问题，**先自己尝试理解**原因，再问 Agent
   105|
   106|你：SonarQube 报了一个"Cognitive Complexity 超标"。
   107|我先看代码自己想想怎么重构，你给我 3 分钟，
   108|然后我告诉你我的方案，你再给反馈。
   109|→ 先自己尝试，再用工具验证
   110|```
   111|
   112|**错误用法**：
   113|
   114|```
   115|你：跑 SonarQube，把所有问题修了。
   116|→ 你跳过了"自己发现问题"和"理解问题本质"两个关键步骤
   117|```
   118|
   119|#### cynical-sally（⭐88）
   120|
   121|```
   122|用途：给代码打分的坦诚审查，适合检验你的代码质量直觉
   123|```
   124|
   125|**正确用法**：
   126|
   127|```
   128|你：这段代码我给自己打 7/10，主要槽点是错误处理不够优雅。
   129|你的评分呢？差异在哪？
   130|→ 先自我评分，再对比。差距就是学习点
   131|```
   132|
   133|#### deliberation（⭐121）
   134|
   135|```
   136|用途：让多个模型对同一问题给出意见，求共识或分歧
   137|```
   138|
   139|**正确用法**：
   140|
   141|```
   142|你：我对这个架构设计有一个结论。
   143|让 3 个不同模型从不同角度挑战它。
   144|我先听听它们的质疑，再决定是否修改。
   145|→ 多模型作为"思考的镜子"，决策权在你
   146|```

   234|## 二、学习型 Skill / 框架推荐
   235|
   236|### 2.1 费曼技巧类（★★★★★）
   237|
   238|#### feynman-tutor（⭐19）
   239|
   240|```
   241|原理：你教 AI，AI 找你的理解漏洞
   242|工作流：
   243|  1. 你选择一个概念
   244|  2. 你用自己的话解释给 Agent
   245|  3. Agent 追问、找矛盾
   246|  4. 你补充/修正
   247|  5. 循环直到 Agent 认为你讲清楚了
   248|```
   249|
   250|**食用指南**：
   251|
   252|```
   253|你：加载 feynman-tutor skill，我要解释 LangGraph 的 State 管理机制。
   254|
   255|第一步：你先自己组织语言，准备解释
   256|第二步：大胆说出来，哪怕不完整
   257|第三步：认真对待 Agent 的每一个追问 —— 如果答不上来，那就是盲区
   258|第四步：把答不上来的点记到 Obsidian，明天复习
   259|```
   260|
   261|#### smart-learn（⭐16）
   262|
   263|```
   264|原理：五步流程 — 概念图 → 费曼解释 → 自测 → 联想 → 总结
   265|特点：生成 Mermaid 思维导图 + Word 实时同步
   266|```
   267|
   268|**食用指南**：
   269|
   270|```
   271|你：加载 smart-learn，我要学 MCP 的 Transport 层。
   272|
   273|流程：
   274|  1. 概念图：自己先画一个脑图框架，再让 skill 帮你完善
   275|  2. 费曼解释：你说给 AI 听
   276|  3. 自测：skill 出题，你回答
   277|  4. 联想：你自己找类比，skill 验证类比边界
   278|  5. 总结：你自己写总结，skill 审查完整性
   279|```
   280|
   281|#### RetainCraft（⭐3）
   282|
   283|```
   284|原理：循证学习科学——间隔重复 + 主动回忆 + 交错学习 + 因果提问
   285|```
   286|
   287|**食用指南**：
   288|
   289|```
   290|适合用作每周复习工具：
   291|  每周日加载 RetainCraft，它会根据间隔重复算法，
   292|  从你过去学过的概念中挑出"该复习了"的知识点，
   293|  用主动回忆的方式考你。
   294|```
   295|
   296|#### cogniloop（⭐2）
   297|
   298|```
   299|原理：拒绝直接给答案的苏格拉底式 AI 导师
   300|特点：你问它问题，它不回答问题，只问你问题
   301|```
   302|
   303|**食用指南**：
   304|
   305|```
   306|当你觉得自己"好像懂了但说不清楚"时：
   307|你：加载 cogniloop，我想理解 Agent 的 ReAct 模式。
   308|    
   309|cogniloop 不会解释 ReAct，它会问：
   310|  "你觉得 Reasoning 和 Acting 之间是什么关系？"
   311|  "为什么需要在它们之间交替而不是顺序执行？"
   312|  "你之前见过类似的设计模式吗？"
   313|
   314|你的任务是回答这些问题——如果你能清晰回答，说明你真懂了。
   315|```

   359|## 三、工具的"错误食用"警示
   360|
   361|### 最容易踩的坑
   362|
   363|| 工具 | 错误用法 | 正确做法 |
   364||---|---|---|
   365|| 任何代码生成 MCP | "帮我写一个完整 Agent" | "我要实现这个函数，我的方案是…… 有什么问题？" |
   366|| 费曼技巧类 | 让 AI 解释概念（这根本不是费曼技巧） | 你解释，AI 追问 |
   367|| SonarQube | 自动修复所有告警 | 自己先 review，再看它的报告做对比 |
   368|| 文件系统 MCP | "帮我搭整个项目结构" | 你自己设计结构，用它加速创建 |
   369|| 笔记 MCP | "帮我整理笔记" | 你自己写，让它补充你遗漏的维度 |
   370|| Playwright MCP | "自动完成这个研究并出报告" | "帮我拿到这 5 页的原文，我自己读" |
   371|
   372|### 一句话检测法
   373|
   374|> **使用工具后，如果你能说出一句"原来这个点我之前没想到"，说明你学到了。**  
   375|> 如果你只能说"工具帮我做完了"，说明你被工具替代了。

   379|## 四、推荐组合工作流
   380|
   381|### 学习新概念（40 分钟）
   382|
   383|```
   384|1. 自己读文档/代码（10 min）
   385|2. 用 smart-learn / feynman-tutor 做费曼测试（15 min）
   386|3. 把盲区记到 Obsidian（5 min）
   387|4. 用 sequential-thinking 深挖一个纠结的点（10 min）
   388|```
   389|
   390|### 编码练习（60 分钟）
   391|
   392|```
   393|1. 自己写代码（25 min）—— 不看文档，凭记忆
   394|2. cynical-sally / SonarQube 做 code review（10 min）
   395|3. 对比差异，理解自己遗漏了什么（15 min）
   396|4. 把学到的记到 basic-memory / Obsidian（10 min）
   397|```
   398|
   399|### 周复习（30 分钟）
   400|
   401|```
   402|1. 闭卷回忆本周学的 3 个核心概念（5 min）
   403|2. RetainCraft 出题测试（10 min）
   404|3. 用 basic-memory 对比"我以为我懂的" vs "实际记录的"（10 min）
   405|4. 把薄弱点加到下周计划（5 min）
   406|```

   446|## 七、最终建议
   447|
   448|> **工具只做 30% 的事：获取信息、执行重复操作、做结构化反馈。**  
   449|> **70% 的事必须是你：定义问题、组织思路、做出判断、输出理解。**
   450|
   451|如果你发现自己持续是"接收者"而非"输出者"，说明工具选错了，或者用法偏了。这时候不是换工具，是换用法——把"帮我做"改成"帮我审"。
